# /plan-feature

Planeje a feature solicitada antes de implementar.

Use quando o usuário pedir:

```text
/plan-feature nome-da-feature
```

Entregue:

1. Objetivo.
2. Escopo.
3. Fora de escopo.
4. Histórias de usuário.
5. Regras de negócio.
6. Entidades afetadas.
7. Telas afetadas.
8. Casos de uso.
9. Testes necessários.
10. Riscos.
11. Plano em passos pequenos.

Não implemente código ainda.

# /write-tests

Crie ou melhore testes para o módulo informado.

Verifique:

1. Caso feliz.
2. Caso de erro.
3. Caso de borda.
4. Cálculo financeiro.
5. Persistência, se aplicável.
6. Regressões possíveis.

Ao final:

- liste testes criados;
- explique o que cada teste garante;
- rode os testes quando possível.

# Checklist de Feature

Antes de considerar uma feature pronta, verificar:

## Produto

- [ ] A feature está no escopo do MVP atual?
- [ ] O objetivo educacional está claro?
- [ ] A linguagem é adequada para jovens?
- [ ] Não há promessa de ganho financeiro?

## Técnica

- [ ] Código está organizado?
- [ ] Regra de negócio está fora da UI?
- [ ] Nomes estão claros?
- [ ] Não há duplicação desnecessária?
- [ ] Não há overengineering?

## Testes

- [ ] Caso feliz testado?
- [ ] Caso de erro testado?
- [ ] Caso de borda testado?
- [ ] Cálculos financeiros testados?

## Documentação

- [ ] Backlog atualizado?
- [ ] Regras de negócio atualizadas?
- [ ] Modelo de dados atualizado?
- [ ] Decisões técnicas registradas?
